# 세상 부수기 업적·경험치·UI 전면 업그레이드 설계

> 작성일: 2026-07-17
>
> 상태: 사용자 권장안 일괄 승인
>
> 제품 범위: 개인·비상업용 모바일 웹게임

## 1. 결정 요약

이번 업그레이드는 기존 파괴 게임 위에 별도 메타게임을 덧씌우는 작업이 아니다. 즉시 부수는 재미를 유지하면서 `행동 발견 → 업적 달성 → 경험치 → 꾸미기 성장 → 다음 목표`가 자연스럽게 이어지도록 진행 구조와 전체 UI를 함께 다시 설계한다.

승인된 핵심 결정은 다음과 같다.

1. 모든 조건과 보상을 처음부터 공개하는 영구 업적 32개를 제공한다.
2. 업적은 쉬움 10개, 보통 10개, 어려움 8개, 달인 4개로 나눈다.
3. 업적별 경험치는 난이도에 따라 50, 100, 200, 400으로 고정한다.
4. 누적 경험치는 인정된 업적 ID의 보상 합으로 계산하며 별도 증감값으로 저장하지 않는다.
5. 레벨은 1부터 20까지이며 공격력이나 확률에 영향을 주지 않는다.
6. 보상은 칭호, 프로필·공유 카드 테두리, 기록책 테마만 제공한다.
7. 기록책을 `홈·업적·꾸미기·설정`으로 구성된 전체 화면 플레이어 허브로 확장한다.
8. 게임, 첫 시작 선택, 프로필 생성·로그인, 기록책을 `한밤의 스티커 아케이드` 시각 언어로 통일한다.
9. 현재 기록으로 증명되는 신규 업적은 자동 소급하되 축하 알림은 한 번으로 묶는다.
10. 오늘의 도전은 기존 도장 보상만 유지하고 경험치를 반복 지급하지 않는다.

## 2. 현재 문제

### 발견성과 동기

- 영구 업적은 5개뿐이라 첫 세션, 중기, 장기 목표의 사다리가 부족하다.
- 업적 5개는 기록책에 처음부터 보이지만 `📖` 아이콘과 `부순 기록`이라는 이름 아래 텍스트 카드로 표시되어 업적 시스템으로 발견하기 어렵다.
- 카드에 아이콘, 난이도, 경험치, 완료율이 없어 무엇을 왜 해야 하는지 한눈에 보이지 않는다.
- 완료 보상은 대표 칭호뿐이라 장기 성장감이 약하다.

### 화면 일관성

- 게임은 짙은 남색 경기장과 노란 충격선을 사용하지만 프로필 화면은 베이지색 전체 화면과 일반 흰 입력 폼을 사용한다.
- 시작 선택과 회원가입이 게임의 일부가 아니라 다른 웹 서비스로 이동한 것처럼 느껴진다.
- 상단 이모지 버튼 5개는 기능은 제공하지만 게임 HUD보다 도구 모음처럼 보인다.
- 기록책은 프로필, 오늘의 도전, 업적, 스킨, 통계, 설정을 한 스크롤에 모두 넣어 업적 32개를 수용할 수 없다.

### 데이터와 동기화

- 클라이언트와 Edge Function이 업적을 정확히 5개로 제한한다.
- XP와 레벨 데이터가 없고 서버·클라이언트 업적 목록이 중복 정의되어 있다.
- 현재 서버는 누적 카운터로 기존 5개 업적을 재확인하지만 클라이언트가 보낸 해금 시각도 병합할 수 있다.
- XP를 직접 더해 저장하면 오프라인 재전송이나 두 기기 동시 달성에서 중복 지급될 수 있다.

## 3. 목표와 제외 범위

### 목표

- 첫 이용자는 10초 안에 전체 업적, 조건, 진행도, 받을 경험치를 찾을 수 있다.
- 첫 2분 안에 쉬운 업적을 최소 1개 달성할 수 있다.
- 첫 세션에는 2~3개 성취가 가능하고 어려움·달인 업적은 여러 세션의 장기 목표가 된다.
- 모든 업적은 다른 플레이 방식을 발견하거나 의미 있는 누적 목표를 제공한다.
- 경험치와 레벨의 의미를 한 문장으로 설명할 수 있다.
- 게임과 프로필 화면이 같은 제품으로 보인다.
- 기존 기록, 선택한 칭호, 스킨, 설정을 잃지 않는다.
- 게스트와 로그인 이용자는 같은 업적·경험치 공식을 사용한다.
- 오프라인과 여러 기기에서도 업적과 경험치가 중복되지 않는다.
- 추가 UI로 인해 첫 공격까지 걸리는 시간과 전투 프레임이 나빠지지 않는다.

### 계속 제외

- 공격력, 피해량, 확률, 콤보 시간 같은 능력치 성장
- 코인, 상점, 에너지, 뽑기, 유료 보상
- 리더보드, 공개 프로필, 친구 비교, PvP
- 연속 출석, 미접속 손실, 퀘스트 실패·감점
- 무한 레벨, 환생, 시즌·주간 패스
- 숨겨진 업적
- 새 캐릭터, 무기, 타겟
- 황금 타겟, FEVER, 공유처럼 새 동기화 카운터가 필요한 업적
- 운영자가 실행 조건이나 XP를 원격 수정하는 규칙 DSL
- 게스트 기록을 새 프로필로 가져오기

## 4. 성장 구조

### 경험치

난이도별 보상은 공개 이후 바꾸지 않는다.

| 난이도 | 업적 수 | 업적당 경험치 | 합계 |
|---|---:|---:|---:|
| 쉬움 | 10 | 50 | 500 |
| 보통 | 10 | 100 | 1,000 |
| 어려움 | 8 | 200 | 1,600 |
| 달인 | 4 | 400 | 1,600 |
| 합계 | 32 | - | 4,700 |

`totalXp`는 저장하지 않고 해금된 업적의 고정 XP를 합산해 계산한다. `level`도 저장하지 않고 아래 경계표에서 파생한다. 업적 ID, 조건, XP는 불변이다. 조건이나 보상이 실질적으로 바뀌면 기존 ID를 수정하지 않고 새 ID를 만든다.

### 레벨

| 레벨 | 누적 XP | 레벨 | 누적 XP |
|---:|---:|---:|---:|
| 1 | 0 | 11 | 1,500 |
| 2 | 50 | 12 | 1,800 |
| 3 | 100 | 13 | 2,100 |
| 4 | 200 | 14 | 2,450 |
| 5 | 300 | 15 | 2,800 |
| 6 | 450 | 16 | 3,200 |
| 7 | 600 | 17 | 3,600 |
| 8 | 800 | 18 | 4,000 |
| 9 | 1,000 | 19 | 4,400 |
| 10 | 1,250 | 20 | 4,700 |

- 최고 레벨은 20이다.
- 레벨 20 뒤에도 완료 업적과 총 XP를 그대로 표시하지만 무한 숫자를 만들지 않는다.
- 오늘의 도전 도장은 XP에 포함하지 않는다.
- 기존 사용자는 현재 누적 기록으로 충족한 업적을 소급 인정받고 같은 공식으로 레벨이 결정된다.

### 꾸미기 보상

- 모든 업적: 고정 경험치
- 대표 칭호 8개: `산산조각`, `끊기지 않는 손`, `무기 도감 완성`, `세계 순환 전문가`, `최애의 한 방`, `기술 박사`, `모든 손의 마무리`, `모든 무기의 달인`
- 레벨 5: `첫 균열` 프로필·공유 카드 테두리
- 레벨 10: `번쩍이는 밤` 테두리와 기록책 색상 테마
- 레벨 15: `폭발하는 노을` 테두리와 기록책 색상 테마
- 레벨 20: `전설의 왕관` 테두리와 기록책 색상 테마
- 기존 캐릭터 스킨과 기능은 다시 잠그지 않는다.
- 잠긴 꾸미기는 필요한 레벨과 다음 행동을 공개한다.

## 5. 영구 업적 카탈로그

각 조건은 이미 저장·동기화되는 누적 데이터만 사용한다. 동일한 단순 카운터의 단계형 업적은 최대 3개까지만 둔다.

### 기존 5개 호환

- `first_destroy`, `charge_master`, `variety_10`, `world_cycle`, `combo_50` ID와 최초 해금 시각은 보존한다.
- 기존 이름은 그대로 유지해 선택된 대표 칭호가 사라지지 않게 한다.
- 현재 기록으로 다른 신규 조건이 충족되면 카탈로그 공개 시각을 해금 시각으로 사용한다.
- 소급 해금 여러 개는 `지난 기록으로 업적 N개를 찾았어요, 경험치 +X` 한 번으로 알린다.

## 6. 핵심 사용자 흐름

### 첫 방문과 프로필

1. 게임 배경과 로고를 유지한 상태에서 시작 선택 카드가 올라온다.
2. `프로필로 이어하기`와 `이 기기에서 바로 놀기`를 같은 크기의 선택 카드로 보여준다.
3. 게스트는 한 번의 선택으로 게임을 시작한다.
4. 프로필 생성은 `ID → PIN → 확인`의 현재 단계가 보인다.
5. 입력 오류는 값을 지우지 않고 해당 필드 아래에서 원인과 다음 행동을 알려준다.
6. 로그인·생성 중에도 남색 경기장 배경을 유지하고 크림색은 읽기 쉬운 종이 카드 표면에만 사용한다.

### 플레이와 첫 업적

1. HUD는 타겟과 파괴 연출을 가리지 않는다.
2. 첫 유효 공격이나 파괴로 업적이 열리면 도장 애니메이션과 `경험치 +50`을 짧게 보여준다.
3. 경험치 조각이 `LV` 기록책 버튼으로 흡수되고 레벨 진행이 잠깐 확장된다.
4. 여러 업적이 동시에 열리면 한 장의 합산 알림으로 묶는다.
5. 알림이 끝나면 전투 화면은 작은 `LV n` 진입점만 남긴다.

### 업적 확인과 꾸미기

1. `LV n` 버튼을 누르면 전체 화면 기록책 홈이 열린다.
2. 홈에서 레벨, 총 XP, 다음 레벨, 전체 완료율, 오늘의 도전, 달성에 가까운 업적 3개를 본다.
3. 업적 화면에서 분야와 `전체·진행 중·완료` 필터를 사용한다.
4. 모든 미완료 업적도 이름, 난이도, 조건, 진행도, XP를 공개한다.
5. 꾸미기 화면에서 획득한 칭호, 테두리, 기록책 테마를 선택한다.
6. 설정 화면에서 강타 방식, 움직임, 진동, 소리를 바꾼다.

## 7. 화면 정보 구조

### 게임 HUD

- 좌측 상단: 현재 연속과 최고 연속을 한 개의 만화 점수판으로 묶는다.
- 우측 상단: `LV n` 기록책 버튼, 소리 버튼, 더보기 버튼만 둔다.
- 더보기: 기록 카드 공유, 다음 타겟, 처음부터를 48px 메뉴 항목으로 제공한다.
- 첫 이용 때 `LV n` 버튼에 `업적 0/32` 안내 꼬리표를 한 번 보여준다.
- 무기 바 이름은 최소 12px로 키우고 선택 항목은 노란 테두리, 만화식 오프셋 그림자, 눌림 반응을 사용한다.
- FEVER 무지개와 전체 화면 연출 예산은 기존처럼 제한한다.

### 기록책 전체 화면 허브

상단에는 닫기, 현재 레벨, 짧은 프로필 상태만 둔다. 주요 화면은 네 개다.

1. `홈`: 레벨·XP, 전체 완료율, 오늘의 도전, 곧 달성할 업적 3개, 최근 보상
2. `업적`: 네 분야 선택, 전체·진행 중·완료 필터, 32개 공개 카드
3. `꾸미기`: 대표 칭호, 프로필·공유 카드 테두리, 기록책 테마, 기존 캐릭터 모습
4. `설정`: 강타 방식, 움직임 줄이기, 진동, 소리, 프로필 관리

업적 카드 읽기 순서는 `아이콘 → 이름 → 난이도 → 조건 → 진행 숫자와 비율 → 경험치 → 상태`다. 단계형 업적은 같은 시리즈로 시각적으로 묶되 각 조건과 보상을 독립적으로 읽을 수 있어야 한다.

### 프로필·가입

- 화면 전체 배경은 게임과 같은 밤 잉크 그라데이션을 유지한다.
- 프로필 패널은 따뜻한 종이 카드, 두꺼운 잉크 윤곽, 2~4px 만화식 그림자를 사용한다.
- 첫 선택 화면에는 깨지는 세상 로고와 짧은 혜택 문구를 사용한다.
- 게임 내부 위험·동기화·컴플라이언스 용어는 사용자에게 노출하지 않는다.
- 게스트 기록이 새 프로필로 이동하지 않는 사실은 생성 시작 지점에서 한 번 정확히 안내한다.
- 로딩 중 버튼 크기는 바뀌지 않고 버튼 내부 문구와 작은 균열 게이지만 변한다.

## 8. 시각 언어: 한밤의 스티커 아케이드

- 쉬움은 종이 스티커, 보통은 금속 핀, 어려움은 금색 균열 메달, 달인은 왕관 도장 형태로 구분한다.
- 난이도는 색뿐 아니라 형태와 텍스트로도 구분한다.
- 사용자 화면에는 이모지 대신 현재 캔버스 그림체와 맞는 가벼운 CSS·SVG 아이콘을 우선 사용한다.

### 모션

- 업적 달성: 0.8초 안에 도장이 찍히고 작은 균열이 퍼진다.
- 경험치 상승: 노란 조각이 기록책 버튼으로 이동한다.
- 레벨 상승: 1초 이내의 스티커 팝과 짧은 화면 강조를 사용한다.
- 프로필 전환: 배경은 유지하고 종이 카드만 올라온다.
- 움직임 줄이기에서는 이동·확대 없이 테두리와 숫자만 바뀐다.

## 9. 데이터와 동기화

### 단일 카탈로그

- 순수 데이터 업적 카탈로그 하나를 클라이언트와 Edge Function이 함께 사용한다.
- 카탈로그는 ID, 이름, 분야, 난이도, 조건, 목표, XP, 칭호 제공 여부, 아이콘 키를 가진다.
- 원격 데이터는 실행 조건을 전달하지 않는다.
- 카탈로그 버전은 2로 올린다.

### 저장 상태

- 현재 `schemaVersion: 1`과 기존 로컬 저장 키는 유지해 불필요한 전체 마이그레이션을 피한다.
- `achievements`는 기존 `{ unlockedAt, seen }` 구조를 유지하고 허용 ID를 32개로 확장한다.
- XP와 레벨은 저장하지 않는다.
- 프로필 설정에 선택한 `frameId`와 `recordBookThemeId`를 추가한다.
- 기존 저장에 새 설정이 없으면 `default`를 사용한다.

### 서버 권위

로그인 이용자의 서버 처리 순서는 다음과 같다.

1. 기기·순서·연산 ID 중복을 기존 방식으로 차단한다.
2. 승인된 누적 카운터 연산을 서버 상태에 적용한다.
3. 서버가 고정 카탈로그로 32개 조건을 다시 계산한다.
4. 조건을 만족한 업적만 서버 수락 시각으로 해금한다.
5. 클라이언트의 해금 시각 주장은 무시한다.
6. `seen`은 서버에서 이미 조건이 충족된 업적에만 병합한다.
7. 총 XP와 레벨을 서버 인정 업적에서 파생한다.
8. 테두리·테마 선택은 필요한 레벨을 서버가 확인한 뒤 반영한다.

게스트는 같은 카탈로그와 조건 평가기를 로컬에서 사용한다. 프로필 로그인 뒤에는 서버 상태가 최종 기준이며 게스트 기록은 합치지 않는다.

### 소급과 호환

- 기존 5개 해금 기록과 가장 이른 신뢰 가능한 서버 시각을 보존한다.
- 신규 업적은 현재 카운터로 다시 계산하고 카탈로그 공개 시각을 해금 시각으로 사용한다.
- 백엔드는 구·신 프론트 연산을 모두 읽는다.
- 구 클라이언트가 보내는 업적 해금 시각은 승인 근거로 사용하지 않는다.
- 새 Edge Function을 먼저 배포하고 기존 프론트가 정상 동기화되는지 확인한 뒤 UI를 연다.
- 롤백 기준은 신구 계약을 모두 읽는 호환 Edge Function이다.
- 새 기능 플래그 행은 추가하지 않고 기존 `gamification_enabled`를 비상 차단으로 사용한다.

## 11. 접근성·모바일 기준

- 핵심 터치 영역은 48px 권장, 최소 44px다.
- 입력 글자는 16px 이상, 무기 이름은 12px 이상이다.
- 320, 360, 390, 430px와 태블릿에서 가로 넘침이 없어야 한다.
- 글자 확대 200%에서 조건, XP, 버튼이 잘리지 않아야 한다.
- 텍스트 대비는 일반 글자 4.5:1, 큰 글자 3:1 이상이다.
- 업적 진행 막대에는 `23 / 50, 46%`처럼 숫자와 비율을 함께 제공한다.
- 탭 전환은 스와이프에 의존하지 않고 명시적인 버튼으로 가능해야 한다.
- 포커스 복원, 포커스 트랩, ESC·닫기, 화면 읽기 순서를 유지한다.
- 소리, 진동, 색, 움직임 중 하나가 꺼져도 달성 상태를 이해할 수 있어야 한다.
- 실제 iOS Safari와 Android Chrome에서 키보드가 열린 채 프로필 생성·로그인을 완료할 수 있어야 한다.

## 12. 성능·분석·운영

### 성능

- 업적 32개는 가상 목록을 사용하지 않고 분야·상태 필터로 범위를 줄인다.
- 업적 DOM은 허브가 열릴 때, 필터가 바뀔 때, 관련 진행·해금이 바뀔 때만 갱신한다.
- 매 프레임이나 파티클 틱에 업적을 평가·렌더링하지 않는다.
- 흐림 효과와 상시 애니메이션을 제한하고 큰 신규 배경 이미지를 사용하지 않는다.

### 분석

기존 개인정보 제한을 유지하며 아래 enum·count 이벤트만 추가한다.

- `achievement_hub_opened`: 진입 위치
- `achievement_filter_changed`: 분야·상태 enum
- `achievement_unlocked`: 업적 ID, 난이도, XP
- `level_reached`: 레벨
- `cosmetic_selected`: 종류와 승인된 ID
- `profile_step_viewed`: 시작, ID, PIN, 완료

프로필 ID, PIN, 사용자 입력, 원시 좌표, 업적 문구는 분석에 넣지 않는다.

### 운영

- 업적 조건과 XP는 코드 카탈로그에서만 변경한다.
- 운영자 화면에는 카탈로그 버전, 난이도별 달성 수, 레벨 분포를 읽기 전용으로 보여준다.
- `gamification_enabled=false`는 업적·XP·레벨·보상 UI와 평가를 닫지만 기존 기록은 보존한다.
- 기능을 다시 열면 저장된 누적 기록으로 업적을 재계산한다.

## 13. 검증과 출시 게이트

### 자동 검증

- 카탈로그가 정확히 32개이며 ID가 중복되지 않음
- 쉬움 10, 보통 10, 어려움 8, 달인 4와 총 4,700 XP
- 같은 단순 카운터의 단계가 3개를 넘지 않음
- 기존 5개 ID·이름·해금 시각·선택 칭호 보존
- 모든 업적 진행 경계값과 레벨 1~20 경계값
- 클라이언트·서버가 같은 카탈로그와 조건 결과를 사용
- 클라이언트 해금 시각을 서버가 신뢰하지 않음
- 오프라인 재전송·두 기기 동시 달성에서 XP 중복 없음
- 구 outbox 연산을 신 서버가 처리
- 잠긴 테두리·테마 선택을 서버가 거절
- 소급 해금 알림과 동시 달성 알림이 정확히 한 번
- 32개 카드 필터, 정렬, 포커스 복원, 화면 읽기 문구
- copy lint, TypeScript, 전체 Vitest, production build, audit

### 수동 검증

- 320·390px와 글자 확대 200%에서 게임·기록책·프로필 오버플로 없음
- 게임 → 시작 선택 → 가입 → 게임의 배경과 시각 언어 연결
- 첫 공격 → 업적 → XP → 레벨 버튼 → 다음 목표 흐름
- 로딩, 빈 상태, 오류, 오프라인, 전체 완료 상태
- 움직임 줄이기, 소리 끄기, 진동 끄기
- 실제 iOS Safari와 Android Chrome의 키보드·스크롤·안전 영역
- 저사양 모바일에서 전투 프레임 저하 없음

### 배포 순서

1. 신구 계약을 모두 읽는 Edge Function과 서버 카탈로그를 먼저 배포한다.
2. 구 프론트의 동기화 성공, projection parse, 5xx 부재를 관찰한다.
3. 새 프론트와 전체 화면 허브를 production 설정 preview에서 검증한다.
4. 실제 모바일 스모크와 PM preview 확인을 마친다.
5. 별도 명시적 production 승인을 받은 뒤 Pages를 배포한다.
6. 실제 URL에서 첫 진입, 게스트, 프로필, 업적, 동기화를 확인하고 오류·지연을 관찰한다.

## 14. 성공 기준

- 사용성 테스트 참여자 전원이 10초 안에 전체 업적을 찾는다.
- 참여자의 90% 이상이 XP가 꾸미기 레벨을 올린다고 설명한다.
- 첫 이용자는 2분 안에 쉬운 업적 1개 이상을 달성할 수 있다.
- 첫 파괴율은 현재 대비 3%p 이상 하락하지 않는다.
- 첫 업적 뒤 기록책 허브 열기 비율은 35% 이상이다.
- 다양한 무기 사용 비율이 증가하고 특정 무기 독점 사용이 감소한다.
- 게임과 프로필 화면이 같은 제품이라는 평가가 90% 이상이다.
- 모바일 오버플로, 44px 미만 핵심 버튼, 대비 위반이 0건이다.
- 동기화된 업적·XP 불일치가 0건이다.

- 기술 관점은 기존 동기화 카운터로 판정 가능한 약 30개와 서버 파생 XP를 추천했다.
- 모든 관점이 단순 XP 숫자, 능력치 보상, 출석 압박, 상점은 반대했다.

### 2라운드 공방

- 디자인의 36개는 풍부하지만 반복 횟수형 업적으로 부풀 위험이 지적됐다.
- 기술의 약 30개는 안전하지만 보상과 전체 화면 허브가 빠지면 사용자 요청을 충분히 충족하지 못한다는 반론이 있었다.
- 경험치는 서버 인정 업적 합, 게스트는 같은 공식의 로컬 합으로 계산하는 데 합의했다.
- 오늘의 도전 XP와 무한 레벨은 반복 압박·동기화 복잡성 때문에 제외하는 쪽으로 수렴했다.

### 3라운드 점수

가중치는 사용자 요청 적합 25%, 즉시 스트레스 해소 정체성 20%, 첫 세션 UX 15%, 장기 풍부함 15%, 기술·데이터 안전 15%, 유지보수성 10%다.

평균 87.4점의 B안을 채택한다. 32개 중 최소 절반은 서로 다른 행동을 유도하고 같은 단순 카운터 단계는 최대 3개로 제한한다. 경험치·레벨은 파생값이며 서버가 조건을 다시 계산한다. 오늘의 도장 반복 XP와 무한 레벨은 명시적으로 거절한다.
