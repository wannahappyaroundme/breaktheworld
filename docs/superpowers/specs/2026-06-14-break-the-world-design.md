# Break The World (세상 부수기) — 디자인 스펙

> 스트레스 해소용, 통쾌하게 "세상"을 와장창 부수는 인터랙티브 웹 애니메이션.
> 작성일: 2026-06-14 · 상태: 설계 확정(사용자 승인 "고!")

---

## 1. 개요 / 목적 (Overview)

스트레스를 받을 때 들어와서 **마음껏 세상을 부수며 스트레스를 푸는** 웹페이지.
잔인하지 않고 **통쾌하고 와장창 깨지는** 느낌. 다양한 파괴 방식(21종)과
귀여운/강력한 캐릭터들이 등장해 질리지 않게 계속 부술 수 있다.
GitHub Actions로 GitHub Pages에 배포해 친한 사람에게 링크로 공유.

- **📱 모바일 전용(mobile-only)** — 폰에서 손가락으로 두드려 부수는 경험에 집중. 데스크톱 별도 대응 없음.
- **비상업 / 지인 공유용** — 저작권 부담 없음(오리지널 SVG 두들 아트로 제작).
- **재미 + 높은 퀄리티**가 핵심. 플래시게임 같은 손맛과 비주얼.

## 2. 타겟 사용자 (Target Users)

- **스마트폰으로** 스트레스 풀고 싶은 일반 사용자(모바일 전용).
- 친한 사람에게 링크 공유 → 폰 브라우저에서 바로 즐김.
- 코드 모르는 PM(프로젝트 오너)도 결과물을 바로 즐기고 공유 가능.

## 3. 아트 방향 (Art Direction)

레퍼런스: 사용자가 첨부한 "솜뭉치 강아지가 위에서 지구를 깔아뭉개 산산조각내는" 손그림.

- **플래시게임 / 손그림 두들 감성**: 까만 손그림 외곽선 + 파스텔 채움 + 노란 스피드·임팩트 라인.
- 지구는 초록·파랑·갈색 덩어리로 쪼개짐. 캐릭터는 표정 있고 귀엽거나 강렬.
- **모든 아트는 SVG로 직접 제작**(코드 작성). 무한 확대 무손실, 색·표정 수정 자유, 비용 0.
- 동적 효과(균열·파편·폭발·빔)는 캔버스 + **rough.js**로 손그림 느낌 렌더.

## 4. 핵심 컨셉 & UX 흐름 (Core UX)

1. 화면 중앙에 부술 **타겟**이 등장(지구).
2. 상단 **무기 선택 바**에서 파괴기 선택(기본값: 망치).
3. 화면을 **탭하면 그 자리가 부서짐**. **쓸기(드래그)하면 연속 타격/긁기**. 멀티터치로 동시 타격.
4. 파괴기 발동 시: 데미지 + 파편 생성 + 파티클(먼지·불꽃) + 화면 진동(shake) + 섬광 + 효과음.
5. 타겟이 완전히 부서지면 잠깐 멈췄다가 **다음 타겟이 자동 등장**(지구→도시→"세상" 글자→반복).
6. **콤보 카운터**로 얼마나 부쉈는지 표시. **리셋/다음** 버튼, **🔇 사운드 토글**.

조작 정리(**터치 기준**): *탭 = 발동 위치, 드래그(쓸기) = 연속, 무기 바 = 무엇으로 부술지.*
시네마틱 캐릭터기(시나모롤 등)는 탭 시 캐릭터가 등장해 화면 전체에 한 방.
손가락이 닿기 쉬운 **세로(portrait) 폰 화면 기준**으로 레이아웃 설계.

## 5. 타겟 (Targets) — 부서지면 자동 교체

| 타겟 | 설명 |
|---|---|
| 🌍 지구 | 초록·파랑·갈색 대륙의 행성. 첨부 그림 스타일로 덩어리 쪼개짐. |
| 🏙️ 도시 스카이라인 | 빌딩들이 무너지고 먼지 폴폴. |
| `세상` 거대 글자 | 한글 "세상"이 유리/돌처럼 산산조각. |

→ 마지막 타겟 부서지면 처음으로 순환(살짝씩 변주: 색/배치).

## 6. 파괴기 (Weapons / Destroyers) — 총 21종

### 물리·원소계 12종 (포인트/드래그형)
1. 🔨 **망치 강타** — 탭 지점 방사형 균열 + 파편 튐 + 진동.
2. 👊 **거대 주먹** — 화면 밖에서 주먹이 날아와 강타, 크레이터.
3. 🧊 **유리 와장창** — 유리처럼 쩍 갈라져 파편 쏟아짐. 드래그로 쭉 긁기.
4. 🔪 **레이저 절단** — 드래그 경로 따라 빔으로 잘리고 조각 분리 낙하.
5. ☄️ **운석 충돌** — 하늘에서 낙하 → 크레이터 + 충격파 링 + 먼지구름.
6. 🚀 **미사일 폭격** — 여러 발 연쇄 폭발.
7. 💣 **대폭발** — 화염구 + 섬광 + 강한 진동 + 파편 사방.
8. ⚡ **번개** — 하늘에서 번개 내리쳐 갈라짐 + 감전 스파크.
9. 🔥 **화염 → 재** — 불태워 재로 스러짐. 드래그로 화염방사.
10. 🌪️ **토네이도** — 회오리가 빨아올려 산산조각.
11. ❄️ **빙결 → 박살** — 얼렸다가 탭하면 쨍! 박살.
12. 🕳️ **블랙홀** — 중력으로 모든 조각이 빨려들어가 소멸.

### 캐릭터계 9종 (시네마틱, 탭 발동)
13. ☁️ **시나모롤 깔아뭉개기** — 위에서 뚝 떨어져 납작하게 짓이김(첨부 그림의 그 장면).
14. 🫰 **타노스 핑거스냅** — 딱! 세상의 절반이 먼지로 흩날려 사라짐.
15. 🦾 **아이언맨 리펄서 빔** — 날아 들어와 빔 발사 → 대폭발.
16. 🟢 **헐크 스매시** — 두 주먹으로 지각을 쾅! 내리쳐 붕괴.
17. 🦖 **고질라 브레스** — 발로 밟고 입에서 광선 브레스 🔥.
18. 🐉 **드래곤볼 에너지파** — 충전 후 거대 에너지파로 관통.
19. 🐱 **귀여운 반전 고양이** — 통통한 고양이가 엉덩이로 깔아뭉갬(귀여운데 파괴적).
20. 🟣 **메타몽 변신 강타** — 흐물흐물 늘어나 거대 주먹/망치로 변신해 내리침.
21. 🍯 **곰돌이 푸 꿀폭탄** — 꿀단지 쏟아 끈적 폭발 + 엉덩이 압살.

> 캐릭터는 오리지널 두들 오마주로 SVG 제작. "딱 알아보게" 닮은꼴.

## 7. 아키텍처 (Modules) — 각 파일 1역할

```
src/
├─ main.ts                 진입점: 캔버스 셋업, 터치 입력 처리, 부팅
├─ engine/
│  ├─ loop.ts              requestAnimationFrame 게임루프(deltaTime), 탭 숨김 시 정지
│  ├─ renderer.ts          캔버스 컨텍스트/레이어, 고해상도(devicePixelRatio) 대응
│  ├─ particles.ts         파티클 시스템(먼지·불꽃·연기·재) + 오브젝트 풀링
│  ├─ shatter.ts           타겟 비트맵을 다각형 조각으로 쪼개기(보로노이식)
│  ├─ camera.ts            화면 흔들림(shake) + 섬광(flash) + 줌 펀치
│  ├─ audio.ts             Web Audio 프로시저럴 효과음 + 마스터 음소거
│  └─ input.ts             터치 입력(탭·쓸기/드래그·멀티터치) → 발동 이벤트
├─ effects/                ★ 공용 이펙트 툴킷(아래 8장)
│  ├─ explosion.ts         화염구·섬광·연기
│  ├─ crack.ts             균열 라인 전파
│  ├─ fragmentBurst.ts     파편 튀어나가기
│  ├─ shockwave.ts         충격파 링
│  ├─ beam.ts              빔/광선
│  ├─ projectile.ts        하늘에서 낙하(운석/미사일)
│  └─ dust.ts              먼지구름
├─ targets/
│  ├─ target.ts            인터페이스(draw/hitTest/takeDamage/isDestroyed/fragments)
│  ├─ earth.ts · city.ts · wordSesang.ts
│  └─ targetManager.ts     자동 교체/순환
├─ weapons/
│  ├─ weapon.ts            인터페이스(apply(point,target,world))
│  ├─ (21종 파일: hammer.ts, glass.ts, meteor.ts ... pooh.ts)
│  └─ weaponBar.ts         상단 무기 선택 UI
├─ assets/                 SVG 아트(타겟·캐릭터) — 직접 제작
├─ ui/                     콤보 카운터 · 사운드 토글 · 리셋/다음
└─ style.css
```

## 8. 공용 이펙트 툴킷 (Effect Toolkit) — 퀄리티 비결

21종을 0부터 만들지 않는다. **재사용 가능한 효과 프리미티브**를 만들고,
각 파괴기는 이를 **조합한 레시피 + 시그니처 비주얼 1개**로 구성.

예) 운석 = `projectile`(낙하) + `shockwave` + `dust` + `crack` + `camera.shake`.
예) 아이언맨 빔 = 캐릭터 등장 + `beam` + `explosion` + `camera.shake`.

→ 적은 코드로 일관된 고퀄, 새 파괴기 추가도 쉬움.

## 9. 데이터 흐름 / 게임 루프 (Data Flow)

```
RAF loop ── update(dt) → render()
입력(탭/쓸기·드래그/멀티터치)
  → 선택된 weapon.apply(point)
     → target.takeDamage + shatter.fragments
     → effects/* 스폰 + particles 스폰
     → camera.shake/flash
     → audio.play
TargetManager: target.isDestroyed → 잠깐 정지 → 다음 타겟 로드
콤보: 타격마다 +1, 일정 시간 무입력 시 리셋
```

## 10. 사운드 (Audio)

- **Web Audio API 프로시저럴 합성** — 외부 음원 파일/라이선스 불필요, 즉시 재생.
  - 폭발/붐: 저역 사인 스윕 + 노이즈 버스트.
  - 유리 와장창: 고역 노이즈 짧은 클릭음 다발.
  - 타노스 스냅: 짧은 클릭음 + 휘잉 + 저역 럼블.
- 우상단 **🔇 토글**(사무실에서도 안전). 첫 사용자 제스처에서 AudioContext 활성화.
- (나중에 실제 음원으로 교체 가능.)

## 11. 모바일 전용 설계 (Mobile-Only) — 핵심

> **데스크톱 별도 대응 없음.** 세로 폰 화면을 기준으로 설계(데스크톱 브라우저로 열려도 동작은 하되 최적화 대상 아님).

- 캔버스가 뷰포트를 꽉 채움. `viewport` 메타로 `user-scalable=no`, 더블탭 줌·스크롤·당겨서새로고침 방지.
- **터치 입력**: touchstart/move/end → 통합 입력으로(탭=발동, 쓸기/드래그=연속, 멀티터치=동시 타격으로 더 통쾌하게).
- **엄지 도달 영역** 고려: 무기 선택 바는 화면 하단 또는 가로 스크롤 + 큰 터치 타깃. 콤보/토글은 상단.
- 세로(portrait) 기준 레이아웃. 가로 모드에서도 깨지지 않게 캔버스 리사이즈 대응.
- 모바일 성능: 파티클 수 자동 스케일(기기 성능/화면 크기 기준), devicePixelRatio 상한, 60fps 목표.
- iOS Safari: 첫 탭 제스처에서 AudioContext 활성화(자동재생 정책 대응).

## 12. 성능 (Performance)

- 오브젝트 풀링(파티클·파편), 화면 밖 컬링, 파티클 총량 캡.
- SVG 아트는 로드 시 오프스크린 캔버스로 **1회 래스터라이즈** 후 스프라이트 재사용
  (rough 손그림 느낌은 비트맵에 구워서 매 프레임 비용 절감).
- shatter는 비트맵 조각을 애니메이션(빠름). 탭 숨김 시 루프 정지.

## 13. 배포 (GitHub Actions → GitHub Pages)

- GitHub 저장소 `breaktheworld` 생성.
- `.github/workflows/deploy.yml`: `push to main` → Node 셋업 → `npm ci` →
  `npm run build` → `dist/` 아티팩트 업로드 → `actions/deploy-pages`로 Pages 배포.
- Vite `base`를 저장소명으로 설정(에셋 경로 보정).
- 결과 라이브 URL: `https://<github-id>.github.io/breaktheworld/`.

## 14. MVP 범위 vs 나중 (Scope)

**MVP(이번에 만들 것):**
- 타겟 3종 자동 순환.
- 파괴기 21종(물리·원소 12 + 캐릭터 9).
- **터치 전용**(탭/쓸기·드래그/멀티터치) 조작, 무기 선택 바, 세로 폰 레이아웃.
- 공용 이펙트 툴킷, 파편/파티클/화면진동/섬광.
- Web Audio 효과음 + 토글, 콤보 카운터, 리셋/다음.
- GitHub Actions → Pages 배포.

**나중(Phase 2+):** 실제 음원, 점수/공유, 캐릭터 추가, 설정 패널, 업적, 통계.

## 15. 검증 계획 (Verification)

- `npm run build` / `tsc --noEmit` 통과.
- `npm run dev` + 브라우저 **모바일 뷰(기기 에뮬레이션/실기기)**에서 각 파괴기 1회씩 시각 확인. 터치(탭·드래그·멀티터치) 동작 확인.
- 빌드 산출물(`dist/`) Pages 배포 후 **실제 폰에서** URL 접속 확인.

## 16. 리스크 / 열린 질문 (Risks)

- 21종 전부 고퀄 = 작업량 큼 → 공용 툴킷으로 완화, 구현 계획에서 단계적으로(툴킷 + 우선 파괴기 → 나머지).
- 캐릭터 SVG 아트 퀄리티 = 핵심 → 실루엣/표정에 공들임, 초기 1~2개로 톤 확정 후 확장.
- 모바일 성능 → 파티클 자동 스케일로 대응.
