# 세상 부수기 게이미피케이션·캐릭터 다양화 설계

> 작성일: 2026-07-16
>
> 상태: 방향 승인 완료, 문서 검토 대기
>
> 제품 범위: 개인·비상업용 모바일 웹게임

## 1. 결정 요약

이번 업그레이드는 화면에 레벨·재화·상점을 추가하는 방식이 아니다. 기존의 즉시 파괴 경험을 유지하면서 아래 세 가지를 하나의 흐름으로 연결한다.

1. 탭은 빠른 기술, 길게 누르기는 충전 강타로 구분한다.
2. 캐릭터는 한 번에 전부 파괴하지 않고, 캐릭터별 부분 기술 2개와 충전 결정타 1개를 가진다.
3. 오늘의 도전 1개와 영구 도장 5개로 다음 행동과 누적 성취를 만든다.

협의회 최종 점수는 압축형 게이미피케이션 86.5점, 조작만 개선 82.2점, 레벨·XP 중심 메타게임 72.3점이었다. 압축형은 즉각 재미와 재방문 이유를 함께 만들면서 화면 과잉을 피한다.

## 2. 현재 문제와 근거

### 입력

- 현재 입력은 `down`과 14px 이상 움직인 `drag`만 구분한다.
- 누른 시간, 손을 뗀 시점, 취소 상태가 없으므로 정지한 롱프레스는 탭 1회와 같다.
- 콤보는 실제 피해와 상관없이 입력마다 증가하므로 빠른 문지르기와 자동 입력에 과도하게 유리하다.

### 캐릭터

- 서로 다른 캐릭터 9종과 시나모롤·메타몽의 예전 버전 2개가 별도 항목으로 등록되어 실제 무기 항목은 23개다.
- 캐릭터 항목 11개 중 타노스를 제외한 10개가 충돌 시 `detachAll`을 호출한다.
- 캐릭터의 그림·색·등장 동작은 다르지만 파괴 결과는 대부분 전체 파괴 하나로 수렴한다.

### 진행과 화면

- 영구 저장은 최고 콤보와 누적 파괴 수 두 값뿐이다.
- 누적 파괴 수는 마일스톤 알림과 공유 카드 외에는 확인하기 어렵다.
- 상단에는 콤보와 버튼 5개, 하단에는 무기 바가 있으며 FEVER, 신기록, 등급, 토스트가 각각 별도 레이어로 표시된다.
- 새 시스템을 상시 패널로 추가하면 타겟보다 UI가 먼저 보일 위험이 있다.

## 3. 목표와 제외 범위

### 목표

- 첫 방문자는 설명을 읽지 않아도 탭, 쓸기, 길게 누르기의 차이를 발견한다.
- 모든 캐릭터는 파괴 위치·형태·단계가 구분되는 고유 기술을 가진다.
- 같은 캐릭터를 반복 사용해도 기술과 파괴 방향이 달라진다.
- 캐릭터 공격은 평균 2~3회 안에 타겟을 완파한다.
- 오늘의 도전과 업적은 선택형 동기이며 플레이 실패나 손실을 만들지 않는다.
- 퀘스트·기능 스위치·기본 통계를 운영자 화면에서 관리할 수 있다.
- 기존 최고 콤보와 누적 파괴 기록을 잃지 않는다.

### 제외 범위

- 플레이어 회원가입과 기기 간 진행 동기화 (이후 `2026-07-16-player-profile-sync-design.md` 승인으로 대체)
- 리더보드, 친구 대전, 경쟁 보상
- 레벨, XP, 코인, 상점, 에너지
- 연속 출석, 미접속 손실, 퀘스트 실패·감점
- 캐릭터·무기·타겟 신규 추가
- 네이티브 앱 전환
- 상업화, 유료 광고, 공개 마케팅을 위한 권리 정비

현재 캐릭터와 개인 공유 기능은 사용자의 개인·비상업용 전제에 따라 유지한다. 제품은 상업화 준비 기능을 범위에 포함하지 않는다.

## 4. 핵심 플레이 흐름

1. 사용자가 화면을 누르면 타겟이 살짝 눌리고 짧은 시각·햅틱 피드백이 온다. 아직 피해는 정산하지 않는다.
2. 0.45초 전에 손을 떼면 탭 기술 1회가 발동한다.
3. 누른 채 16px 이상 움직이면 충전 후보가 취소되고 기존 쓸기 공격으로 전환된다.
4. 0.45초 이상 유지하면 얇은 원형 충전 표시와 무기별 전조가 나타난다.
5. 0.45~1.1초 사이에 손을 떼면 충전량에 비례한 강타가 1회 발동한다.
6. 1.1초에 최대 충전 상태가 되고, 손을 떼면 무기 결정타가 발동한다.
7. 충전 중에는 콤보 만료를 유예하고, 강타 1회만 콤보와 진행도에 반영한다.
8. 실제 파편이 하나 이상 분리된 공격만 유효 공격으로 기록한다.
9. 타겟 파괴, 충전 성공, 무기 다양성 이벤트가 오늘의 도전과 업적을 갱신한다.
10. 완료 알림은 한 번에 하나만 표시되고, 보상은 별도 수령 버튼 없이 자동 반영된다.

## 5. 입력 규칙

### 상태

입력은 `idle → pressed → tap | dragging | charging → released | cancelled` 상태로 관리한다.

| 상태 | 진입 조건 | 결과 |
|---|---|---|
| `pressed` | 첫 손가락 입력 | 눌림 피드백만 표시 |
| `tap` | 0.45초 전 해제, 이동 16px 미만 | 빠른 공격 1회 |
| `dragging` | 이동 16px 이상 | 경로를 따라 제한 빈도로 공격 |
| `charging` | 0.45초 이상 유지 | 충전량과 전조 표시 |
| `released` | 탭 또는 충전 후 정상 해제 | 공격 1회 정산 |
| `cancelled` | `touchcancel`, 화면 숨김, 리셋, 다음, 타겟 교체 | 피해·콤보·퀘스트 정산 없음 |

### 세부 정책

- 첫 번째 활성 포인터만 충전할 수 있다. 추가 손가락은 탭과 쓸기만 처리한다.
- 쓸기 콤보는 거리뿐 아니라 시간 간격 하한을 적용해 문지르기 파밍을 제한한다.
- 충전 중 콤보는 현재 입력이 끝난 뒤 0.2초까지 유예한다.
- 최대 충전은 자동 반복하지 않는다. 손을 떼야 한 번 발동한다.
- 롱프레스가 어려운 사용자는 설정에서 `두 번 탭으로 강타`를 선택할 수 있다.
- `prefers-reduced-motion`에서는 큰 흔들림과 줌을 줄이고 충전 표시·숫자·소리로 상태를 전달한다.
- 소리와 진동이 꺼져 있어도 충전 완료를 화면으로 확인할 수 있다.

## 6. 무기별 충전 동작

### 물리·원소 무기 12종

- 기존 탭·쓸기 효과를 유지한다.
- 충전 강타는 같은 무기 레시피를 사용하되 피해 범위, 힘, 효과 크기를 충전량에 따라 키운다.
- 최대 충전은 기본 범위의 약 1.6배, 유효 피해 예산은 최초 파편 수의 55~70%를 상한으로 한다.
- 매 프레임 파티클을 추가하는 자동 연타 방식은 사용하지 않는다.
- 블랙홀처럼 기존에 전체 파괴하던 시네마틱 무기도 충전량에 따른 부분 흡수로 바꾸고 최대 3행동 완파 규칙을 따른다.

### 캐릭터 무기 9종

각 캐릭터는 탭 기술 2개와 최대 충전 결정타 1개를 가진다.

| 캐릭터 | 탭 기술 A | 탭 기술 B | 최대 충전 결정타 |
|---|---|---|---|
| 시나모롤 | `구름콩콩`: 탭 중심 원형 압착 | `긴귀 휩쓸기`: 가로 띠 파괴 | `하늘폭신 프레스`: 넓은 타원 압착 |
| 타노스 | `보석 튕김`: 작은 영역 4곳 소멸 | `중력 움켜쥠`: 바깥 조각을 중심으로 흡수 | `운명의 스냅`: 넓은 범위 디졸브 |
| 아이언맨 | `손바닥 리펄서`: 탭 지점 폭발 | `가슴빔 절단`: 탭 방향 직선 절단 | `연속 리펄서 폭격`: 여러 지점 연쇄 타격 |
| 헐크 | `주먹 쿵`: 탭 지점 원형 분쇄 | `지각 발구르기`: 하단 띠 파괴 | `천둥 스매시`: 중심 분쇄와 방사형 균열 |
| 고질라 | `꼬리 휩쓸기`: 가로 띠 파괴 | `발자국 짓밟기`: 좌우 한쪽 원형 압착 | `원자 숨결`: 대각선 광선 파괴 |
| 에너지파 | `기탄 연사`: 작은 영역 5곳 타격 | `순간이동 타격`: 서로 떨어진 3곳 타격 | `초에너지파`: 굵은 직선 관통 |
| 고양이 | `앞발 툭툭`: 작은 영역 3곳 타격 | `꼬리 싹쓸이`: 곡선 띠 파괴 | `궁디 쿵`: 넓은 원형 압착 |
| 메타몽 | `말랑 펀치`: 탭 지점 탄성 분쇄 | `쭉쭉 롤러`: 세로 또는 가로 띠 파괴 | `만능변신 강타`: 직전 원소 무기 패턴 복제, 없으면 망치 |
| 곰돌이 푸 | `꿀단지 철퍽`: 여러 조각을 붙였다 낙하 | `배통통 밀기`: 한 방향으로 밀어냄 | `꿀폭탄 깔아뭉개기`: 넓은 압착과 꿀 파편 |

### 피해와 랜덤 정책

- 탭 기술은 최초 파편 수 기준 약 35~50%를 파괴한다.
- 충전 강타는 충전량에 따라 최초 파편 수 기준 약 55~80%를 파괴한다.
- 새 타겟을 최대 충전 한 번으로 바로 100% 파괴하지 않는다.
- 공격 뒤 남은 파편이 최초 파편의 20% 이하거나 같은 캐릭터의 세 번째 유효 공격이면 시그니처 마무리로 완파한다.
- 남은 파편이 있으면 최소 1개는 반드시 분리해 타노스의 잔여 1개 고착 문제를 없앤다.
- 탭 기술 A/B는 시드 기반 제한 랜덤으로 선택하며 같은 기술이 3회 연속 나오지 않는다.
- 기술 안의 방향, 충돌 위치, 보조 파편 모양도 시드로 변주한다.
- 탭한 위치가 공격 시작점에 반영되어 사용자의 조작감을 유지한다.
- `actionId`와 `targetRunId`가 일치할 때만 비동기 충돌의 피해를 정산한다. 이미 교체된 타겟에는 늦은 연출이 피해를 주지 않는다.

### 기존 모습 정리

- 시나모롤과 메타몽의 예전 버전은 별도 무기에서 제거한다.
- 두 모습은 각 캐릭터의 `기본`과 `클래식` 모습으로 저장한다.
- 사용 횟수, 퀘스트, 업적은 모습이 아니라 캐릭터 ID에 귀속한다.
- 무기 바는 물리·원소 12종과 캐릭터 9종, 총 21개로 정리한다.

## 7. 오늘의 도전과 영구 도장

### 오늘의 도전

- 하루에 한 개만 활성화한다.
- KST 오전 4시를 날짜 경계로 사용한다.
- 당일 처음 배정된 퀘스트는 카탈로그가 바뀌어도 그날 동안 유지한다.
- 미완료 손실, 연속 출석, 만료 경고, 감점은 없다.
- 완료 시 파괴 도장 1개를 자동 지급한다.

초기 기본 카탈로그는 아래 세 가지다.

| ID | 화면 문구 | 완료 조건 |
|---|---|---|
| `charged_finisher_2` | 꾹 와장창 2번 | 최대 충전 강타 2회 |
| `characters_3` | 캐릭터 3종 만나기 | 서로 다른 캐릭터 3종 사용 |
| `targets_3` | 타겟 3개 부수기 | 유효 파괴 3회 |

황금 타겟처럼 무작위 출현에 의존하는 조건과 단순 탭 횟수 조건은 사용하지 않는다.

### 영구 도장 5개

| ID | 이름 | 조건 |
|---|---|---|
| `first_destroy` | 첫 와장창 | 타겟 1개 파괴 |
| `charge_master` | 꾹 와장창 장인 | 최대 충전 강타 누적 10회 |
| `variety_10` | 골고루 파괴 | 서로 다른 무기 10종 사용 |
| `world_cycle` | 세상 한 바퀴 | 세상·지구·도시를 각각 1회 이상 파괴 |
| `combo_50` | 콤보 폭주 | 최고 콤보 50 달성 |

### 보상

- 도장, 칭호, 공유 카드 테두리만 제공한다.
- 공격력, 확률, 콤보 시간 같은 플레이 능력치는 보상으로 사용하지 않는다.
- 사용자가 획득한 칭호 하나를 선택해 공유 카드에 표시할 수 있다.
- 보상 수령 버튼은 없으며 완료 시 자동 반영한다.

## 8. 화면 정보 구조

### 기본 플레이 화면

- 화면 중심은 항상 타겟과 파괴 연출이 차지한다.
- 기존 상단 `✨` 버튼을 `📖 기록책` 버튼으로 교체한다. 새 상단 버튼은 추가하지 않는다.
- 버튼은 최소 44px 실제 `button` 요소와 한글 접근성 이름을 사용한다.
- 첫 유효 공격 뒤 오늘의 도전 진행을 4초 동안 한 줄로 보여준 뒤 접는다.
- 이후 진행 알림은 50% 도달과 완료 시점에만 다시 보여준다.
- 기록책 버튼에는 미확인·완료 상태를 작은 점과 체크 모양으로 표시한다.

### 기록책 하단 시트

탭 UI를 여러 개 만들지 않고 한 장의 스크롤 시트로 구성한다.

1. `오늘의 도전`: 문구, 진행 숫자, 완료 도장
2. `부순 기록`: 영구 도장 5개와 다음 조건
3. `캐릭터 모습`: 시나모롤·메타몽 기본/클래식 선택
4. `내 기록`: 최고 콤보, 누적 파괴, 충전 강타, 사용한 무기 수

잠긴 도장은 조건을 숨기지 않고 사용자가 다음에 할 행동을 쉬운 문장으로 보여준다.

### 알림과 연출 예산

- 큰 전체 화면 연출은 FEVER, 신기록, 최대 충전 결정타에만 사용한다.
- 퀘스트 진행은 숫자와 짧은 채움으로 표현한다.
- 도장 완료는 남색·노랑 계열의 1색 도장 애니메이션만 사용한다.
- 무지개는 FEVER 전용으로 유지한다.
- 알림 큐 우선순위는 `신기록 > 영구 도장 > 오늘의 도전 > 일반 안내`다.
- 한 번에 하나의 알림만 표시하고 같은 알림은 합친다.
- 업데이트 안내는 매 방문이 아니라 버전당 한 번만 자동 표시한다.

### 사용자 화면 문구 원칙

- `COMBO`는 `연속`, 기록책 내부에서는 `최고 연속`으로 표현한다.
- 퀘스트, 업적, XP 같은 시스템 용어 대신 `오늘의 도전`, `부순 기록`, `도장`을 사용한다.
- 막힌 상태는 다음 행동으로 안내한다.
- 사용자 화면 문구에는 em dash를 사용하지 않는다.
- 카피 린트가 금지어와 em dash를 검사한다.

## 9. 시스템 구조

### 모듈 경계

```text
Input
  → ActionController
      → Weapon / CharacterMoveSet
          → DamageResult
              → GameEventBus
                  ├─ Combo / FEVER
                  ├─ ProgressStore
                  ├─ QuestEngine / AchievementEngine
                  ├─ NotificationQueue
                  └─ AnalyticsClient
```

- `Input`: 누름·이동·해제·취소만 해석한다. 게임 보상은 모른다.
- `ActionController`: 탭·쓸기·충전 상태와 동시에 하나의 시네마틱만 관리한다.
- `Weapon`과 `CharacterMoveSet`: 시각 연출과 피해 프로필을 정의한다.
- `DamageResult`: 분리된 파편 수, 공격 출처, 캐릭터, 기술, 타겟 실행 ID를 반환한다.
- `GameEventBus`: 검증된 결과를 콤보·진행·알림·분석에 전달한다.
- `ProgressStore`: 버전형 로컬 상태, 검증, 마이그레이션, 저장 실패 폴백을 담당한다.
- `QuestEngine`과 `AchievementEngine`: 순수 이벤트 reducer로 진행도를 계산한다.
- `NotificationQueue`: 여러 축하 이벤트가 겹치지 않게 순서를 정한다.
- `AnalyticsClient`: 허용된 enum·카운트만 전송하고 실패가 게임을 막지 않게 한다.

### 로컬 진행 상태

키는 `btw.progress.v1` 하나를 사용한다.

```ts
interface ProgressStateV1 {
  schemaVersion: 1
  catalogVersion: number
  installSeed: string
  lifetime: {
    validHits: number
    chargedFinishers: number
    totalTargets: number
    bestCombo: number
    stamps: number
  }
  byWeapon: Record<string, { uses: number; finishes: number; seenMoves: string[] }>
  byTarget: Record<string, { destroys: number }>
  achievements: Record<string, { unlockedAt: string; seen: boolean }>
  daily: {
    dayKey: string
    questId: string
    target: number
    progress: number
    completedAt: string | null
    stampAwarded: boolean
  }
  profile: {
    selectedTitle: string | null
    skins: Record<string, string>
    strongInput: 'hold' | 'doubleTap'
    reducedMotion: boolean
    haptics: boolean
  }
}
```

- 로드 시 음수, 무한대, 알 수 없는 ID, 손상 JSON을 검증한다.
- 잘못된 부분만 안전한 기본값으로 복구하고 게임은 계속 실행한다.
- 기존 `btw.bestCombo`와 `btw.totalTargets`는 새 상태 저장이 성공한 뒤 마이그레이션한다.
- 프레임과 충전 틱마다 저장하지 않는다. 공격 종료, 타겟 파괴, 도장 해제, 설정 변경, `pagehide`에서 저장한다.
- `localStorage`가 막혀 있으면 메모리 모드로 계속 플레이하고 한 번만 안내한다.

### 이벤트

진행도에 사용할 이벤트는 아래 enum으로 제한한다.

- `ATTACK_RESOLVED`
- `CHARGE_RELEASED`
- `TARGET_DESTROYED`
- `WEAPON_USED`
- `FEVER_STARTED`
- `SHARE_COMPLETED`

각 이벤트는 `actionId`, `targetRunId`, `source`, 실제 분리 파편 수를 포함한다. `source=demo|system` 이벤트는 퀘스트와 업적에서 제외한다. 원시 좌표, 사용자 문구, 프롬프트, 개인정보는 저장하거나 전송하지 않는다.

## 10. 운영자 페이지와 서버

### 원칙

- 플레이어는 로그인하지 않는다.
- 진행 기록과 모습 선택은 기기에 저장한다.
- Supabase는 운영 설정, 익명 집계, 운영자 로그인에만 사용한다.
- 원격 연결이 실패해도 내장 퀘스트 카탈로그와 로컬 진행으로 게임이 계속된다.

### 운영자 기능

- ID/PW 로그인
- 운영자 계정 조회·활성화·비활성화
- 오늘의 도전 생성·수정·활성화·비활성화
- 도전 조건, 목표 수치, 노출 기간 관리
- `gamification_enabled`, `character_variants_enabled`, `analytics_enabled` 기능 스위치
- 방문, 첫 유효 공격, 첫 파괴, 충전 사용, 도전 완료, 공유 완료의 일별 집계
- 캐릭터별 사용 수, 평균 완파 행동 수, 충전 성공·취소율

### 데이터 모델

| 테이블 | 용도 | 주요 필드 |
|---|---|---|
| `admin_users` | 운영자 권한 | `user_id`, `role`, `active` |
| `quest_catalog` | 도전 카탈로그 | `id`, `copy`, `event_type`, `target`, `active_from`, `active_to`, `enabled`, `version` |
| `feature_flags` | 운영 기능 스위치 | `key`, `enabled`, `updated_at` |
| `analytics_events` | 제한된 익명 이벤트 | `event_type`, `day_key`, `install_hash`, `weapon_id`, `value`, `created_at` |
| `analytics_daily` | 운영자 통계 집계 | `day_key`, `event_type`, `weapon_id`, `count` |

### 보안과 장애 격리

- Supabase 공개 키만 브라우저에 두고 서비스 역할 키는 서버 환경에만 둔다.
- 운영자 권한은 `admin_users`와 RLS로 중앙 검사한다.
- 운영자 계정 존재 여부가 드러나지 않는 동일한 로그인 실패 응답을 사용한다.
- 익명 이벤트는 Edge Function이 enum, 숫자 범위, 허용된 ID를 검증한다.
- 설치별 행동 버킷과 일일 상한으로 중복·자동 입력 전송을 제한한다.
- 이벤트 전송은 최대 3회 지수 백오프 후 포기하며 플레이 요청은 항상 성공한다.
- 원격 퀘스트는 하루 동안 캐시하고 장애 시 내장 기본값으로 폴백한다.
- 관리자 페이지와 운영 스위치는 게임 캔버스 코드와 별도 진입점으로 분리한다.

## 14. 협의회 기록

### 1라운드 독립 의견

- 사업·제품: 대표 제스처와 재방문 목표가 필요하지만 로비·상점·손실 시스템은 피해야 한다.
- UX·브랜드: 충전 규칙은 하나로 유지하고 화면에는 진행 한 줄만 잠깐 노출해야 한다.
- 게임 디자인: 탭, 쓸기, 충전이 서로 다른 선택이어야 하며 충전 중 콤보 손해가 없어야 한다.
- 데이터·기술: 실제 피해 이벤트, 버전형 저장소, 최대 3행동 완파, 중복 정산 방지가 선행되어야 한다.
- 보안·QA: 로컬 보상은 비경쟁·장식형이어야 하며 입력 취소·타겟 교체·잔여 1개를 테스트해야 한다.
- 마케팅·그로스: 첫 행동을 방해하지 않으면서 충전 발견률·도전 완료율·재방문을 측정해야 한다.

### 2라운드 공격과 절충

- 자동 연타 롱프레스는 콤보 악용과 화면 폭주 위험 때문에 기각했다.
- `pointerdown` 즉시 피해와 충전 결정타를 함께 쓰면 한 행동이 이중 정산되므로 피해는 해제 시 한 번만 정산한다.
- 이번 판 과제 3개와 업적 8~12개는 체크리스트 압박이 커서 오늘의 도전 1개와 업적 5개로 줄였다.
- 완전 랜덤은 캐릭터 정체성과 테스트 재현성을 해치므로 2개 기술과 내부 방향·위치의 제한 랜덤을 사용한다.
- 전체 파괴 제거만 적용하면 잔여 파편 고착이 생기므로 최대 3행동 완파를 보장한다.
- 예전 캐릭터를 별도 무기로 두지 않고 모습으로 통합해 무기 바를 21개로 정리한다.

### 3라운드 점수

| 안 | 평균 점수 | 결론 |
|---|---:|---|
| 조작만 개선 | 82.2 | 즉각 재미는 높지만 재방문 이유가 약함 |
| 압축형 게이미피케이션 | 86.5 | 즉각 재미, 절제, 재방문의 균형이 가장 좋음 |
| 레벨·XP 중심 메타게임 | 72.3 | 스트레스 해소가 숙제로 변할 위험이 큼 |

최종 선택은 압축형 게이미피케이션이다.
