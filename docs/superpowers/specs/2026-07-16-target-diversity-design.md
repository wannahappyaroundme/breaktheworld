# 세상 부수기 타겟 다양화 설계

- 작성일: 2026-07-16
- 상태: 추천안 확정, 구현 전
- 제품 전제: 개인·비상업용, 모바일 전용, 귀엽고 비폭력적인 스트레스 해소 게임
- 한 줄 경험: 손안의 물건부터 태양까지, 부술수록 규모가 커지는 12단 와장창

## 1. 결정

기존 `세상`, `도시`, `지구` 3종을 유지하고 신규 9종을 더해 총 12종으로 확장한다. 첫 사이클은 작은 일상 사물에서 우주로 커지는 순서를 고정한다. 두 번째 사이클부터 작은 사물 구간만 설치 시드로 제한 셔플한다.

아트는 `정적 생성 베이스 이미지 + 코드 피해 마스크 + 코드 파편·물리 + 코드 폴백`의 하이브리드 방식을 사용한다. 런타임 이미지 생성, 외부 이미지 CDN, 미리 만든 파괴 단계 이미지는 사용하지 않는다.

## 2. 목표와 성공 기준

### 목표

- 설명 없이도 타겟 규모가 점점 커지는 흐름을 이해한다.
- 다음 타겟을 보고 싶은 마음이 반복 플레이 동기가 된다.
- 21개 무기의 탭·쓸기·충전이 모든 타겟에서 같은 피해 계약을 사용한다.
- 생성 이미지가 없어도 동일한 타겟을 완성도 있게 플레이한다.
- 신규 이미지가 첫 화면과 저사양 모바일 성능을 늦추지 않는다.

### 성공 기준

- 총 12개 안정 ID와 중복 없는 순환 순서
- 첫 화면은 현재와 다음 타겟 아트만 준비
- 파편 1개가 남아도 다음 유효 공격으로 피해 발생
- 9캐릭터 × 12타겟이 최대 3개 유효 행동 안에 완파
- 이미지 강제 실패 시 12개 타겟 모두 코드 폴백으로 진행
- 50회 전환 뒤 활성 효과·캔버스·힙이 초기 기준 ±10% 안으로 복귀
- 390×844 저사양 Android 기준 p95 프레임 20ms 이하, 50ms 초과 프레임 1% 미만

## 4. 순환 규칙

### 첫 사이클

표의 1~12 순서를 그대로 사용한다. 태양 다음에는 세상으로 돌아간다.

### 두 번째 사이클 이후

- `[alarm, mug, parcel]` 안에서만 결정론적으로 셔플한다.
- `[vending, small-house]` 안에서만 결정론적으로 셔플한다.
- `word`, `city`, `mountain`, `moon`, `earth`, `ringed-planet`, `sun` 순서는 고정한다.
- 시드는 `installSeed + cycleIndex`를 사용한다.
- 같은 시드와 사이클은 같은 순서를 만든다.

### 다음·초기화·재접속

- 타겟을 파괴하거나 `다음`을 누르면 현재 위치를 저장한다.
- `다음`은 파괴 기록, 도전, 도장, 분석에 포함하지 않는다.
- `초기화`는 `word`로 돌아가며 누적 기록과 `cycleIndex`는 유지한다.
- 재접속 시 저장된 타겟을 온전한 상태로 다시 시작한다. 부분 피해는 복원하지 않는다.
- 태양 다음으로 이동할 때 `cycleIndex`를 올리고 다음 순서를 만든다.
- 기존 `세상 한 바퀴` 도장은 `word`, `city`, `earth` 파괴 조건을 유지한다.

## 5. 시스템 구조

### 타겟 정의

```ts
interface TargetDefinition {
  id: TargetId
  name: string
  tier: 'object' | 'place' | 'world' | 'space'
  fragmentCount: number
  assetUrl: string | null
  drawMask(ctx: CanvasRenderingContext2D, size: TargetSize): void
  drawFallback(ctx: CanvasRenderingContext2D, size: TargetSize): void
}
```

카탈로그가 ID, 이름, 이미지, 크기, 파편 수, 피해 마스크, 코드 폴백을 한곳에서 소유한다. 한국어 이름과 배열 위치를 저장 키로 사용하지 않는다.

### 이미지 로더

- 현재 타겟과 다음 타겟, 최대 2개만 디코드한다.
- 첫 타겟은 기존 코드 아트이므로 이미지 준비가 첫 행동을 막지 않는다.
- 첫 입력 뒤 또는 브라우저 idle 시간에 다음 이미지 하나를 준비한다.
- 새 타겟 생성 시 준비된 `ImageBitmap`을 넘긴다.
- 타겟 교체 시 사용하지 않는 비트맵은 `close()`하고 LRU에서 제거한다.
- 데이터 절약 모드에서는 해당 타겟 진입 직전에만 요청한다.

### 피해 마스크와 파편

- 베이스 이미지 알파를 게임 규칙으로 직접 사용하지 않는다.
- 각 타겟의 코드 `drawMask`가 파괴 가능한 본체를 정의한다.
- 파편별 3×3 고정 샘플 중 3점 이상이 마스크 안이면 파편을 유지한다.
- 알람 다리, 머그 손잡이, 행성 고리, 태양 코로나 같은 장식은 코드 레이어로 분리한다.
- 실제 균열, 분리, 중력, 황금 효과, 무기별 피해는 공통 `Breakable.applyDamage()`가 담당한다.
- 피해와 기록은 실제로 분리된 파편 수가 1개 이상일 때만 발생한다.

### 폴백 순서

1. 준비된 정적 WebP를 사용한다.
2. 로드·디코드·검수 실패 시 같은 타겟의 코드 폴백을 즉시 사용한다.
3. 폴백 파편 생성도 실패하면 기록 없이 다음 타겟으로 이동한다.
4. 한 사이클 전체가 실패하면 `세상` 코드 폴백을 유지한다.
5. 기술 오류 팝업은 표시하지 않고 보조기기 알림으로 한 번만 안내한다.

## 6. 생성 이미지 제작 규칙

### 대상

신규 9종인 알람시계, 머그컵, 택배 상자, 자판기, 작은 집, 산, 달, 고리 행성, 태양의 온전한 베이스 아트를 생성한다.

### 공통 화풍

- 귀여운 손그림 장난감 모형 느낌
- 4~6색, 굵은 남색 외곽선, 좌상단의 부드러운 광원
- 화면 점유율 65~85%, 중앙 정렬, 충분한 바깥 여백
- 단일 연결 실루엣, 사실적 질감·연기·폭발·상처 없음
- 문자, 로고, 워터마크, 얼굴, 배경 소품, 바닥 그림자 없음
- 1024px 기준 24px보다 얇은 돌출부 없음
- 내부 구멍은 투명 대신 어두운 채움으로 표현

### 생성·후처리

- 기본 내장 이미지 생성 도구로 대상별 한 장씩 만든다.
- 완전히 평평한 크로마키 배경을 사용하고 로컬 제거 도구로 알파를 만든다.
- 투명 모서리, 피사체 점유율, 가장자리 번짐, 흰 테두리, 키 색 잔여를 검사한다.
- 생성 원본을 그대로 출고하지 않고 팔레트·외곽선·광원을 기준 시트에 맞춰 검수한다.
- 기존 에셋을 덮어쓰지 않고 새 안정 파일명으로 저장한다.

### 파일 예산

- 일반 원본: 1024×1024
- 산·고리 행성: 최대 1536×1024
- 최종: 투명 lossless WebP
- 정사각형 파일당 180KB 이하
- 가로형 파일당 240KB 이하
- 신규 9개 합계 1.8MB 이하
- 디코드된 현재+다음 타겟 아트 합계 12MB 이하
- 생성 모델, 날짜, 최종 프롬프트, 후보 폐기 이유를 내부 제작 기록에 남긴다.

## 7. 저장과 마이그레이션

`btw.progress.v1`에 아래 필드를 최초 구현부터 포함한다.

```ts
targetRotation: {
  catalogVersion: number
  cycleIndex: number
  position: number
  orderIds: TargetId[]
}
```

- 기존 `btw.totalTargets`와 `word`, `city`, `earth` 기록을 보존한다.
- 신규 타겟 기록은 0에서 시작한다.
- 손상, 중복, 누락, 알 수 없는 ID가 있는 `orderIds`는 현재 카탈로그와 시드로 재생성한다.
- 순환 복구는 누적 파괴 기록과 도장 상태를 수정하지 않는다.
- 타겟 카탈로그 변경과 `catalogVersion` 상승은 같은 커밋으로 배포한다.
- 원격 타겟 CRUD와 타겟별 원격 분석은 추가하지 않는다.

## 8. UX와 접근성

- 새 상단 버튼, 타겟 선택 화면, 로비를 추가하지 않는다.
- 현재의 `다음`과 `초기화` 흐름을 유지한다.
- 타겟 생성 시 `현재 타겟: <이름>`을 `aria-live="polite"`로 한 번 알린다.
- 타겟 이름은 쉬운 한국어로 표시하고 이미지 파일명이나 기술 용어를 노출하지 않는다.
- 모션 감소에서는 낙하 반동, 화면 흔들림, 태양 코로나 움직임을 줄인다.
- 색이 없어도 실루엣으로 달, 지구, 태양을 구분한다.
- 이미지 준비 실패는 플레이를 막지 않으며 팝업을 추가하지 않는다.

## 12. 다직군 협의회 기록

### 1라운드 독립 의견

- 제품·게임: 12종 혼합 상승형이 다음 장면 기대와 기존 즉시 플레이를 가장 잘 결합한다.
- UX·브랜드: 생명체와 실제 재난을 피하고 장난감 모형 화풍과 실루엣 차이를 유지해야 한다.
- 프론트엔드·게임 아트: 생성 이미지는 베이스에만 쓰고 피해 마스크와 파편은 코드로 분리해야 한다.
- 데이터·보안: 안정 ID, 버전형 순환, 같은 출처 정적 에셋, 런타임 생성 금지가 필요하다.
- QA·접근성·성능: 252개 자동 조합, 코드 폴백, 현재+다음 2개 디코드, 이름 알림이 필요하다.

### 2라운드 공격과 절충

- 12종 고정 순서의 지루함은 작은 구간만 제한 셔플하고 우주 상승 순서는 고정해 해결한다.
- 좋아하는 타겟 대기 문제는 기존 `다음`과 순환 위치 저장으로 해결하며 선택 UI는 만들지 않는다.
- 원형 우주 타겟의 유사성은 비대칭 달, 대륙 띠 지구, 코드 코로나 태양으로 구분한다.
- 생성 화풍 편차는 공통 프롬프트, 기준 시트, 팔레트·외곽선 검수로 줄인다.
- 이미지·알파 실패는 별도 코드 마스크와 완성된 코드 폴백으로 격리한다.
- 252개 수동 검사는 자동 속성 테스트로 옮기고 수동 검사는 위험 기반 매트릭스로 줄인다.
